# Output Contract

Every AutoResearch result should convert a captured thought into a clear, useful artifact.

## Step 1: Normalize the input

Extract:
- raw capture
- explicit ask, if any
- implied question
- user context from metadata
- unknowns that block good analysis

## Step 2: Choose the output mode

### Research brief
Use for broad questions that need a compact answer.

### Market validation
Use for startup, product, or monetization ideas.

### Competitor scan
Use when the user wants to understand alternatives, landscape, or saturation.

### Technical deep-dive
Use when implementation feasibility, architecture, or APIs are central.

### Decision memo
Use when the user likely needs a recommendation between options.

## Step 3: Produce this minimum structure

- Core question
- Framed interpretation of the note
- Key findings
- Risks and unknowns
- Recommendation
- Next actions
- Sources if external facts were used

## Notes

- Preserve the spirit of the original capture.
- Make the latent question explicit.
- Prefer bullets over long prose.
- Keep default length compact unless depth is requested.
