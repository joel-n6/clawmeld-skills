# ClawMeld Growth

Optimize for revenue, not feature count.

## Workflow

1. Define the proposed feature, product move, or market hypothesis in one sentence.
2. Score the opportunity using `references/icp-scorecard.md`.
3. Run simple revenue math using `references/mrr-math.md`.
4. Identify the bottleneck stage:
   - acquisition
   - activation
   - conversion
   - retention
5. Recommend one of these outcomes:
   - ship now
   - test manually first
   - reframe for a narrower ICP
   - defer
   - kill
6. If proceeding, use `assets/templates/growth-experiment.md` or `assets/templates/go-no-go-review.md`.

## Rules

- Prefer concrete commercial outcomes over generic usefulness.
- Force one ICP unless there is strong evidence for a second.
- Reject vague positioning like "for everyone with ideas."
- Do not recommend feature work unless it improves acquisition, activation, conversion, retention, or willingness to pay.
- If the path to revenue depends on unproven behavior change, recommend a manual or concierge test first.
- Always show the simplest plausible path to $10k/month.

## Revenue-first framing

When evaluating ClawMeld, prefer outcome-based wedges such as:
- voice dump to research brief
- voice note to publish-ready content
- voice capture to structured tasks/follow-ups
- voice note to CRM or client update
- voice note to founder operating system

Avoid treating "voice capture" alone as the product unless the user explicitly wants commodity utility software.

## Output format

Return answers in this structure unless the user asks otherwise:
1. Thesis
2. ICP
3. Offer / promise
4. Revenue math
5. Biggest risk
6. Recommendation
7. Next experiment
